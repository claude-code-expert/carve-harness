01 / 개념

에이전트 하네스란 무엇인가

모델은 지능을 제공하고, 하네스는 실행 환경을 제공한다. OpenHarness는 이 등식을 코드로 증명한다.

Harness = Tools + Knowledge + Observation + Action + Permissions
LLM 단독으로는 텍스트만 생성한다. 파일을 읽고(도구), 도메인 지식을 참조하고(스킬), 결과를 관찰하고, 행동하고, 안전 경계 안에서 움직이게 만드는 것 — 이 전체가 하네스다.

03 / 심장

The Agent Loop

하나의 루프, 무한히 조합 가능. 모델은 무엇을 할지 정하고, 하네스는 어떻게(안전하게·효율적으로·관찰 가능하게)를 처리한다.

openharness/engine — 의사 코드

while True:
    response = await api.stream(messages, tools)

    if response.stop_reason != "tool_use":
        break  # 모델이 작업 완료

    for tool_call in response.tool_uses:
        # Permission check → Hook → Execute → Hook → Result
        result = await harness.execute_tool(tool_call)

    messages.append(tool_results)
    # 루프 지속 — 모델이 결과를 보고 다음 행동 결정
도구 실행 한 번에 권한 검사 → PreToolUse 훅 → 실행 → PostToolUse 훅 → 결과의 파이프라인이 끼어든다. 이 지점이 바로 carve-harness가 프로젝트별 제약(flight-rules)과 검증 훅을 주입하는 자리다.
04 / 매핑

서브시스템 → 하네스 3기둥

강의의 "하네스 3기둥" 프레임으로 OpenHarness를 재배열하면, 각 서브시스템의 역할이 선명해진다.

Pillar 1

제약 (Constraints)

permissions/ · hooks/
무엇을 하면 안 되는지. 멀티레벨 권한 모드, 경로 규칙, 커맨드 차단, PreToolUse 차단 훅(exit code 2).

Pillar 2

피드백 루프 (Feedback)

skills/ · coordinator/ · tools/(검색·테스트)
결과를 관찰하고 다음을 결정. 스킬 로딩, 서브에이전트 검증·조율.

Pillar 3

상태 관리 (State)

memory/ · tasks/ · prompts/ · config/
세션을 넘어 맥락 유지. MEMORY.md, 백그라운드 태스크, 컨텍스트 압축.

05 / 확장

4개 확장점 — 이게 전부다

OpenHarness는 딱 4가지로만 확장된다. carve-harness가 "깎아 생성"하는 산출물도 정확히 이 4종이다.

확장점형태역할carve 생성물
ToolBaseTool 상속 (Pydantic)새 기능 추가. 등록 즉시 JSON Schema 자동 생성으로 모델이 학습.프로젝트 전용 도구
SkillSKILL.mdon-demand 도메인 지식. 호출 시점에만 컨텍스트 로드(토큰 절약)..claude/skills/
Pluginplugin.json + 폴더commands·hooks·agents·MCP 패키지화. claude-code 호환.(번들 배포)
Hookcommand·http·prompt·agent라이프사이클 이벤트 가로채기. blocked=True면 도구 차단..claude/hooks/

커스텀 도구 — 약 10줄이면 끝

class MyToolInput(BaseModel):
    query: str = Field(description="Search query")

class MyTool(BaseTool):
    name = "my_tool"
    description = "Does something useful"
    input_model = MyToolInput

    async def execute(self, args, ctx):
        return ToolResult(output=f"Result for: {args.query}")

커스텀 스킬 — ~/.openharness/skills/my-skill.md

---
name: my-skill
description: Expert guidance for my specific domain
---

# My Skill
## When to use
Use when the user asks about [your domain].

07 / 적용

carve-harness 관점에서

OpenHarness는 carve-harness의 베이스 레퍼런스다. 무엇을 참조하고 무엇을 차용할지 정리한다.

OpenHarness → carve-harness 매핑

10+ 서브시스템 분류
→
designer의 하네스 슬롯 설계 기준 (P2)
4개 확장점
→
generator가 깎아 생성하는 산출물 4종 (P4)
Permission 3단계 + path_rules
→
프로젝트별 flight-rules.md (P5)
PreToolUse / PostToolUse 훅
→
결정적 검증 훅 (P5)
Agent Loop 권한·훅 파이프라인
→
제약·검증 주입 지점
SKILL.md / plugin.json 포맷
→
생성 자산의 호환 포맷
핵심 차용 전략: carve-harness는 OpenHarness를 vendor/에 읽기 전용으로 두고 분류 체계·패턴만 참조한다. OpenHarness가 활발히 변경 중(stars·forks 빠르게 증가, 최근 활동 수일 내)이므로 반드시 핀 버전으로 고정한다.
