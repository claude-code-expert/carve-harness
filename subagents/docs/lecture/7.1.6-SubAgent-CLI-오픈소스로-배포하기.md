# 7.1.6 SubAgent CLI 오픈소스로 배포하기

## CLI 설치 스크립트 · GitHub Releases · npm 배포

> 이전 장에서는 에이전트와 훅을 만들었다.
> 이 장에서는 **이 결과물을 다른 사람이 한 줄로 설치할 수 있게 배포**하는 방법을 다룬다.

---

## 학습 목표

1. **install.sh 설계** — 로컬·원격·언인스톨을 하나의 스크립트로
2. **원라인 설치** — `curl | bash` 패턴과 보안 고려사항
3. **GitHub Releases 자동화** — 태그 푸시 → tar.gz 빌드 → 릴리스 생성
4. **SHA256 체크섬 검증** — 다운로드 무결성 보장
5. **CI 파이프라인** — shellcheck + 자동 테스트
6. **npm 배포** — bash 기반 도구를 npm으로 등록하는 방법
7. **라이선스·기여 가이드** — 오픈소스 운영 기본기

---

## 1. 배포 채널 비교

| 방법 | 장점 | 단점 | 적합한 경우 |
|------|------|------|------------|
| **curl \| bash** (원라인) | 한 줄 설치, 의존성 없음 | 보안 우려, 업데이트 수동 | 셸 스크립트 도구, 데모 |
| **GitHub Releases** | 버전 관리, 체크섬, 변경 이력 | git 클론보다 단계 복잡 | 정식 릴리스 채널 |
| **git clone + install** | 소스 검토 가능 | git 필요, 단계 많음 | 기여자, 검토자 |
| **npm/pnpm/yarn** | `npm i -g` 글로벌 설치, 자동 업데이트 | Node.js 의존성, 패키지 검토 비용 | JS/TS 생태계, CLI 도구 |
| **Homebrew** | macOS 표준, brew update | tap 등록·formula 작성 부담 | macOS 타겟 도구 |

> Squad Agent는 **셸 스크립트 본질**이라 ① + ② 조합을 채택. npm 배포 방식은 후반부에서 다룬다.

## 2. install.sh 전체 구조

```bash
#!/bin/bash
set -euo pipefail

REPO="claude-code-expert/subagents"
RELEASE_BASE="https://github.com/$REPO/releases"

# VERSION 파일 우선, 없으면 fallback
VERSION=$(cat "$(dirname "${BASH_SOURCE[0]}")/VERSION" 2>/dev/null || echo "1.3.1")

AGENTS_DIR="$HOME/.claude/agents"
COMMANDS_DIR="$HOME/.claude/commands"
HOOKS_DIR="$HOME/.claude/hooks"

main() {
  case "${1:-}" in
    --uninstall|-u) uninstall ;;
    --version|-v)   echo "Squad Agent v$VERSION" ;;
    --dev)          register_precommit ;;
    --help|-h)      print_help ;;
    *)              banner; do_install "$(find_source_dir)" || download_and_install ;;
  esac
}

main "$@"
```

> **단일 진입점 main 함수**로 감싸는 이유: `curl | bash`로 다운로드 도중 끊겨도 부분 실행 방지

## 3. 원라인 설치 패턴 — `curl | bash`

```bash
curl -sL https://raw.githubusercontent.com/claude-code-expert/subagents/main/install.sh | bash
```

### 동작 흐름

```
1. curl이 install.sh 전체를 다운로드
2. bash가 stdin으로 받아 실행
3. install.sh가 로컬 소스 디렉토리를 찾음 (find_source_dir)
4. 로컬 소스 없음 → GitHub Releases에서 최신 tar.gz 다운로드
5. SHA256 체크섬 검증
6. ~/.claude/{agents,commands,hooks}/ 에 복사
7. settings.json에 훅 자동 등록 (jq 사용)
```

### 보안 고려
- `set -euo pipefail` — 에러 즉시 종료
- 모든 함수를 `main()`으로 감싸 다운로드 미완료 시 부분 실행 차단
- HTTPS 강제, 체크섬 검증

## 4. 로컬 소스 자동 감지

```bash
find_source_dir() {
  local script_dir
  if [ -n "${BASH_SOURCE[0]:-}" ] && [ -f "${BASH_SOURCE[0]}" ]; then
    script_dir="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
    if [ -d "$script_dir/agents" ] && [ -d "$script_dir/commands" ]; then
      echo "$script_dir"
      return 0
    fi
  fi
  return 1
}
```

### 의도
- `git clone` 받은 디렉토리에서 `bash install.sh` 실행 → **로컬 소스로 설치**
- `curl | bash`로 실행 → 로컬 소스 없음 → **원격 다운로드**
- 동일한 스크립트가 두 모드를 자동 분기

## 5. 원격 다운로드 + 체크섬 검증

```bash
# 최신 릴리스 태그 조회 (HTTP 302 redirect의 location 헤더 파싱)
latest_tag=$(curl -sI "$RELEASE_BASE/latest" | grep -i '^location:' \
             | sed 's|.*/||' | tr -d '\r\n')

# 태그 형식 검증 (vX.Y.Z만 허용)
if [[ ! "$latest_tag" =~ ^v[0-9]+\.[0-9]+\.[0-9]+$ ]]; then
  red "Error: Invalid release tag format: $latest_tag"
  exit 1
fi

archive_url="$RELEASE_BASE/download/$latest_tag/squad-agents-${latest_tag}.tar.gz"
checksum_url="${archive_url}.sha256"

curl -sL "$archive_url" -o "$_TMPDIR/squad.tar.gz"
curl -sL "$checksum_url" -o "$_TMPDIR/squad.tar.gz.sha256"

# SHA256 검증
expected=$(awk '{print $1}' < "$_TMPDIR/squad.tar.gz.sha256")
actual=$(shasum -a 256 "$_TMPDIR/squad.tar.gz" | awk '{print $1}')
[ "$expected" = "$actual" ] || { red "Checksum failed!"; exit 1; }
```

## 6. settings.json 훅 자동 등록 (jq 사용)

```bash
register_hook() {
  local settings="$HOME/.claude/settings.json"
  [ ! -f "$settings" ] && echo '{}' > "$settings"

  if ! command -v jq &>/dev/null; then
    yellow "jq not found — register hooks manually."
    return 0
  fi

  local new_hook='[{"matcher":"","hooks":[{"type":"command",
                  "command":"zsh ~/.claude/hooks/subagent-chain.sh"}]}]'

  # 이미 등록되어 있는지 확인 → 없으면 추가
  if ! jq -e '.hooks.SubagentStart' "$settings" &>/dev/null; then
    local tmp=$(mktemp)
    jq --argjson hook "$new_hook" '.hooks.SubagentStart = $hook' "$settings" > "$tmp"
    mv "$tmp" "$settings"
  fi
  # SubagentStop, UserPromptSubmit 동일 패턴
}
```

> **idempotent**: 여러 번 실행해도 안전. 이미 등록된 훅은 덮어쓰지 않음.

## 7. 언인스톨 지원

```bash
uninstall() {
  yellow "Uninstalling Squad Agent..."

  for name in "${SQUAD_AGENTS[@]}"; do
    [ -f "$AGENTS_DIR/${name}.md" ] && rm -f "$AGENTS_DIR/${name}.md"
  done

  for name in "${SQUAD_COMMANDS[@]}"; do
    [ -f "$COMMANDS_DIR/${name}.md" ] && rm -f "$COMMANDS_DIR/${name}.md"
  done

  for name in "${SQUAD_HOOKS[@]}"; do
    [ -f "$HOOKS_DIR/$name" ] && rm -f "$HOOKS_DIR/$name"
  done

  yellow "Note: .bak backup files were preserved."
  yellow "Note: settings.json hooks require manual removal."
}
```

```bash
# 사용
bash install.sh --uninstall
```

> 백업 파일(`.bak`)은 보존 — 사용자의 커스터마이징 보호

## 8. GitHub Releases 자동화 — `.github/workflows/release.yml`

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'                      # v1.0.0 같은 태그 푸시 시 트리거

permissions:
  contents: write                  # 릴리스 생성 권한

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Read version
        id: version
        run: echo "version=${GITHUB_REF_NAME}" >> "$GITHUB_OUTPUT"

      - name: Build archive
        run: |
          mkdir -p dist/squad-agents
          cp -r agents commands hooks install.sh VERSION LICENSE README.md \
                dist/squad-agents/
          cd dist
          tar czf "squad-agents-${{ steps.version.outputs.version }}.tar.gz" \
                  squad-agents/
          shasum -a 256 "squad-agents-*.tar.gz" > "squad-agents-*.tar.gz.sha256"
```

## 12. pre-commit 훅 (개발자용)

```bash
bash install.sh --dev
```

```bash
register_precommit() {
  local repo_root=$(git rev-parse --show-toplevel)
  local hook_file="$repo_root/.git/hooks/pre-commit"

  cat > "$hook_file" << 'HOOK'
#!/bin/bash
set -euo pipefail
REPO_ROOT="$(git rev-parse --show-toplevel)"
echo "Running squad-router tests..."
bash "$REPO_ROOT/tests/test-router.sh" --quiet || exit 1
bash "$REPO_ROOT/tests/test-files.sh" --quiet || exit 1
echo "All tests passed."
HOOK
  chmod +x "$hook_file"
}
```

> 기여자는 commit 시점에 자동으로 145건 테스트가 돌아 깨진 코드 진입 차단

## 15. npm 배포 — 왜 고려하는가

### 셸 설치(curl | bash)의 한계
- 비기술 사용자에게 "curl 명령을 터미널에 입력하세요"는 진입장벽이 높다
- 자동 업데이트 메커니즘이 없다 — 수동 재설치 필요
- Windows 사용자는 WSL 없으면 불편
- 패키지 매니저 표준 채널이 아님

### npm 배포의 장점
- `npm i -g squad-agents` 한 줄
- `npm update -g squad-agents`로 업데이트
- npmjs.com에 공식 페이지 — 검색 노출
- semver 자동 관리

### 단점
- Node.js 18+ 필요 (셸 도구 대비 무거움)
- 패키지 사이즈가 작아도 등록·관리 비용
- npm 보안 사고(npm-leaked tokens) 대응 필요

## 18. CLI 엔트리포인트 — `bin/squad-agents.js`

```javascript
#!/usr/bin/env node
import { install, uninstall } from '../src/installer.js';
import { fileURLToPath } from 'node:url';
import path from 'node:path';

const __dirname = path.dirname(fileURLToPath(import.meta.url));
const ASSETS_DIR = path.join(__dirname, '..', 'assets');

const args = process.argv.slice(2);
const cmd = args[0] || 'install';

switch (cmd) {
  case 'install':
    await install(ASSETS_DIR);
    break;
  case 'uninstall':
    await uninstall();
    break;
  case '--version':
  case '-v':
    const pkg = await import('../package.json', { assert: { type: 'json' } });
    console.log(`Squad Agent v${pkg.default.version}`);
    break;
  default:
    console.log('Usage: squad-agents [install|uninstall|--version]');
}
```

## 19. installer.js — 핵심 로직

```javascript
import fs from 'node:fs/promises';
import path from 'node:path';
import os from 'node:os';

const HOME = os.homedir();
const TARGETS = {
  agents:   path.join(HOME, '.claude/agents'),
  commands: path.join(HOME, '.claude/commands'),
  hooks:    path.join(HOME, '.claude/hooks'),
};

export async function install(assetsDir) {
  for (const [type, target] of Object.entries(TARGETS)) {
    await fs.mkdir(target, { recursive: true });
    const src = path.join(assetsDir, type);
    const files = await fs.readdir(src);

    for (const file of files) {
      const srcPath = path.join(src, file);
      const dstPath = path.join(target, file);

      // 백업
      try { await fs.copyFile(dstPath, dstPath + '.bak'); } catch {}

      await fs.copyFile(srcPath, dstPath);
      if (type === 'hooks') await fs.chmod(dstPath, 0o755);
      console.log(`  Installed ${file}`);
    }
  }
  await registerHooks();
}
```

## 20. registerHooks.js — JSON 직접 조작

```javascript
import fs from 'node:fs/promises';
import path from 'node:path';
import os from 'node:os';

const SETTINGS = path.join(os.homedir(), '.claude/settings.json');

export async function registerHooks() {
  let config = {};
  try {
    config = JSON.parse(await fs.readFile(SETTINGS, 'utf-8'));
  } catch { /* 파일 없음 → 빈 객체 */ }

  config.hooks ??= {};

  const HOOKS = {
    UserPromptSubmit: 'bash ~/.claude/hooks/squad-router.sh',
    SubagentStart:    'zsh ~/.claude/hooks/subagent-chain.sh',
    SubagentStop:     'zsh ~/.claude/hooks/subagent-chain.sh',
  };

  for (const [event, command] of Object.entries(HOOKS)) {
    if (config.hooks[event]) continue;  // 이미 등록됨
    config.hooks[event] = [{
      matcher: '',
      hooks: [{ type: 'command', command }],
    }];
  }

  await fs.writeFile(SETTINGS, JSON.stringify(config, null, 2));
}
```

> bash + jq의 의존성을 Node 내장 모듈만으로 대체

| 라이선스 | 특징 | 적합한 경우 |
|---------|------|------------|
| **MIT** | 가장 자유, 짧음 | 단순 도구, 라이브러리 |
| **Apache 2.0** | 특허 명시, 변경 고지 | 기업 친화, 패치 가능 |
| **GPL v3** | 카피레프트 | 파생물도 오픈소스 강제 |
| **AGPL v3** | 서버 사이드도 GPL 적용 | SaaS 형태 보호 |

- 자유로운 사용·수정·배포
- 변경 사항 명시 의무
- 특허 보호 조항 (기여자가 특허 소송 시 라이선스 자동 종료)
- 상표권 사용 제한

> Claude Code 생태계 도구는 Apache 2.0이 기본 선택지

## 24. CONTRIBUTING.md — 기여 가이드

```markdown
# Contributing

## Adding a New Agent

1. agents/squad-{name}.md — 에이전트 정의
2. commands/squad-{name}.md — 슬래시 커맨드
3. commands/squad.md — 통합 커맨드 목록 갱신
4. install.sh — SQUAD_AGENTS 배열에 추가

### 필수 섹션
- ## Rules (NEVER 절 최소 1개)
- ## Allowed Commands + ## NEVER Run (Bash 도구 시)
- ## Boundaries — Will/Will Not (Write 도구 시)
- ## Output Format
- description에 `Pipeline:` 라인

## PR 가이드라인
1. PR당 하나의 에이전트
2. agents/와 commands/ 두 파일 모두 포함
3. bash install.sh 로컬 테스트
4. README 에이전트 테이블 갱신
5. Conventional Commits 사용
```

## 25. 버전 관리 전략 — Semantic Versioning

```
MAJOR.MINOR.PATCH
```

| 단계 | 의미 | Squad Agent 예시 |
|------|------|-----------------|
| **MAJOR** | 호환성 깨짐 | v1 → v2: 에이전트 인터페이스 변경 |
| **MINOR** | 기능 추가 (호환) | 1.2.0 → 1.3.0: Squad Router 훅 추가 |
| **PATCH** | 버그 수정 | 1.3.1 → 1.3.2: shellcheck 수정 |

### CHANGELOG.md 형식 (Keep a Changelog)

```markdown
## [1.3.2] - 2026-04-11

### Added
- Automated test suite (145 cases)
- Pre-commit hook (--dev)

### Changed
- CI pipeline replaced with test-router.sh + test-files.sh

### Fixed
- shellcheck SC2221/SC2222 — glob pattern ordering
```
