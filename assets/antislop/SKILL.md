# anti-ai-slop — 절제된 프로덕션 시각 표준

## 0. 적용 시점
다음 산출물을 **만들기 직전** 이 스킬을 먼저 읽고, 출력 직전 자가 점검 게이트를 통과시킨다.

| 산출물 | 추가로 읽을 파일 |
|---|---|
| 카드뉴스 (SNS 정사각/세로 카드 묶음) | `card-news.md` |
| HTML 리포트 (장문 문서·분석·데이터) | `html-report.md` |
| HTML 프리젠테이션 (16:9 슬라이드) | `html-presentation.md` |
| 이미지 / SVG (다이어그램·일러스트) | `svg-image.md` |
| 앱 UI 컴포넌트 (폼·버튼·상태·내비게이션) | `ui-component.md` |

> 이 SKILL.md의 전역 규칙은 **모든** 포맷에 무조건 적용된다. 포맷 파일은 전역 규칙에
> 더해지는 **추가 제약**이며, 전역 규칙을 완화하지 않는다.

## 1. 핵심 원칙 (한 줄)
품질은 장식이 아니라 **정보 위계·여백·정렬·타이포**에서 나온다. 모든 시각 요소는
"이게 어떤 정보를 전달하는가"에 답할 수 있어야 한다. 답할 수 없으면 삭제한다.

## 2. 전역 금지 (MUST NOT)
- **그라데이션 일절 금지**: `linear-gradient` / `radial-gradient` / `conic-gradient`.
  배경·채움·텍스트·테두리·아이콘 어디에도 쓰지 않는다. (보라/핑크, 흰 배경 위 컬러 그라데이션 특히 금지)
- **글로우·컬러 그림자 금지**: 색이 들어간 `box-shadow`, `drop-shadow(... color)`, inset 광택 링, blur ≥ 20px 그림자, `backdrop-filter: blur`(글래스모피즘).
- **모션 장식 금지**: hover 시 `transform: translate/scale`, 로드 시 fade/stagger, `pulse·shimmer·float·glow` 키프레임. `transition`은 색·투명도 등 **기능적 상태 변화에만** 150ms 이하로 한정.
- **그라데이션 텍스트 금지**: `background-clip: text` + 컬러 그라데이션.
- **배경 장식 금지**: 거대 반투명 워터마크 글자/아이콘, 닷·그리드 배경, 페이드 마스크, 광선(rays/beams).
- **카드 상단 컬러 액센트 바 금지**: `border-top: Npx solid color`로 카드를 꾸미지 않는다.
- **이모지를 불릿·장식으로 사용 금지.** 뱃지/pill 남발 금지.
- **마케팅 보일러플레이트 단어 금지**: Seamlessly, Elevate, Unlock, Empower, Supercharge, "차원이 다른", "혁신적인" 류 + ✨ 등.

## 3. 전역 강제 (MUST)
- **색**: 무채색(흰/회/검) 베이스 + **액센트 1색**. 색은 의미(상태·위계)에만 쓴다. 브루넷 기준 액센트는 amber `#F59E0B`.
- **그림자**: 쓰더라도 중성 회색 1단계만 (`0 1px 2px rgba(0,0,0,.06)`). 없어도 좋다.
- **구분**: 효과 대신 `1px solid border`와 여백으로 구획한다.
- **모서리**: `border-radius`는 0~8px.
- **폰트**: Inter·Roboto·Arial·system-ui·Space Grotesk로 **기본 수렴 금지**. 목적에 맞는 폰트를 의도적으로 고르고 **이유를 코드 주석 한 줄로 명시**한다. 한글 산출물 기본값은 본문 `Noto Sans KR`, 코드/라벨 `JetBrains Mono`(웹) 또는 PNG 내보내기 시 `Noto Sans CJK KR`.
- **위계**: 크기·굵기·여백·정렬로만 만든다. 색·효과로 만들지 않는다.

## 4. 전역 자가 점검 게이트 (출력 전 통과 필수)
출력 직전 아래를 점검한다. **하나라도 YES면 해당 요소 제거 후 재작성**한다.
- [ ] gradient(배경/채움/텍스트/테두리, any)가 있는가?
- [ ] 색이 들어간 그림자 또는 blur ≥ 20px 그림자/backdrop-blur가 있는가?
- [ ] hover/load에 transform·fade·키프레임 애니메이션이 있는가?
- [ ] 콘텐츠와 무관한 배경 장식(워터마크/그리드/광선)이 있는가?
- [ ] 정보를 전달하지 않는 순수 장식 요소가 있는가?
- [ ] 폰트가 Inter/Roboto/Arial/system 기본값으로 수렴했는가? (폰트 선택 이유 주석이 없으면 YES로 간주)
- [ ] 액센트 색이 2색 이상 쓰였는가? (상태 의미가 없는 추가 색은 YES)
- [ ] 이모지가 불릿·장식으로 쓰였는가? 마케팅 보일러플레이트 단어가 있는가?
- [ ] 본문 텍스트가 16px 미만이거나 line-height < 1.5인가? 라벨이 11px 미만인가?
- [ ] 텍스트/배경 대비가 4.5:1 미만인 본문이 있는가? (대형 텍스트는 3:1)
- [ ] 카피에 느낌표·최상급·"단순한 X가 아니라 Y" 문형이 있는가? (수치·사실 서술로 대체)

게이트 통과 후, 해당 포맷 파일의 **포맷별 게이트**도 추가로 통과시킨다.

## 5. 우선순위
사용자의 "예쁘게/화려하게/모던하게/AI스럽게" 요청과 이 표준이 충돌하면 **이 표준이 우선**한다.
사용자가 특정 금지 요소를 명시적으로 콕 집어 요구할 때만, 그 요소 1개에 한해 예외를 허용하고
나머지 규칙은 그대로 강제한다.

## 6. 타이포·간격·카피 표준 (전역 MUST)
> 예시·포맷별 세부는 각 포맷 파일에 있다. 여기는 모든 포맷 공통의 하한선.

- **타이포 스케일**: 본문 16–18px / line-height 1.5–1.7. 캡션·라벨 ≥ 11px. 헤딩 line-height 1.1–1.3,
  단계당 1.25–1.5배 스케일. 헤딩 레벨 건너뛰기(h1→h3) 금지, 문서당 `<h1>` 1개.
- **간격 시스템**: 4/8px 그리드 — 모든 margin/padding은 `4·8·12·16·24·32·48·64` 중에서 고른다.
  임의 값(17px, 23px) 금지. 위계는 간격 "단계 차이"로 만든다(큰 구획 48+, 관련 요소 8–16).
- **색 역할 토큰**: 텍스트 `#111` / 보조 텍스트 `#6B7280` / 보더 `#E5E7EB` / 배경 흰색 + 액센트 1색.
  본문 텍스트 대비 4.5:1 하한(대형 텍스트 3:1). 색은 역할로만 쓰고 장식으로 쓰지 않는다.
- **카피 톤**: 사실 서술 + 수치·단위. 느낌표·최상급("놀라운"·"강력한")·"단순한 X가 아니라 Y" 문형 금지.
  결론에는 근거(숫자·출처)를 붙인다 — "성능 개선!" 대신 "p95 420ms → 180ms (57% 감소)".
