# workflow — 절차적 완수 규율

다단계·고모호·검증이 중요한 작업을 "장기 실행형 작업자"로 끌고 가는 절차. 단발 답변·즉흥 진행을 막는다.
기존 자산(`iterate`·`sprint-contract`·`evaluation-criteria`)을 *지휘*한다 — 같은 규칙을 다시 쓰지 않고 호출한다.

## 7단계 루프

1. **목표 재정의** — 요청을 측정 가능한 형태로 바꾼다. 모호하면 핵심 가정을 명시하고 진행, 치명적 모호성은 멈추고 묻는다. (`sprint-contract.md` 목표 절)
2. **작업 분해** — 2단계 이상이면 분해하고 단계별 완료 기준·의존성을 적는다. (`squad-plan`에 위임 가능)
3. **완료 기준** — "무엇이 완료의 증거인가"에 먼저 답한다(테스트·실행 결과·로그). (`evaluation-criteria.md` MUST PASS·DoD 참조)
4. **가정 정리** — 가정·검증법·fallback을 표로 외화한다. (`sprint-contract.md`의 `## 가정` 절)
5. **실행** — 즉시 실행한다. 약속만 하고 미루지 않는다.
6. **자체 검증** — 산출물을 실제로 실행·관찰한다. 코드는 테스트, 문서·시각물은 렌더 확인, 설정은 적용 검증. 에러는 무시하지 않고 추적한다. green 수렴은 `iterate`, 테스트 실행은 `squad-qa`에 위임 가능.
7. **리스크 + 다음** — 남은 불확실성과 "검증되지 않음"을 정직하게 표시하고, 다음 액션이나 에스컬레이션을 제안한다.

## 원칙 (설치된 베이스라인이 단일 출처)

`.claude/CLAUDE.md`의 다음 원칙을 작업 내내 적용한다 — 여기선 포인터만 둔다:
- **Contractual Distrust** — 입력·출력·미래의 나를 의심한다. 경계 4종(정상·매핑·None·변조)을 테스트한다.
- **Externalized Decisions** — 결정에 ID(D1·D2…)를 부여하고, 주석에 이유·비용·탈출구를 적는다.
- **Scope Convergence** — "안 건드릴 것"을 먼저 적어 변경 반경을 좁힌다(갈아엎기보다 얹기).

## 최소 출력 형식

작업 보고는 8요소로 고정한다:
`[목표] [계획] [가정] [실행] [증거] [검증] [리스크] [다음]`
사고 과정·헤징("내 생각엔"·"아마도")은 설치된 Response Control(R1–R6)을 따라 드러내지 않는다. 여기선 형식만 정한다.

## 완료 게이트

7단계가 전부 통과하기 전에는 "완료"를 선언하지 않는다. 게이트 기준은 `evaluation-criteria.md`(MUST PASS)와 `sprint-contract.md`(완료 조건)를 따른다 — behavior·test·문서 현행화·검증까지가 한 단위다.

## 에스컬레이션

다음은 합리적 가정으로 풀 수 없으니 멈추고 보고한다:
- 스펙 밖 결함, 고정 답이 없는 열린 창의 디테일, 지시되지 않은 자발적 깊이, 치명적 모호성.

## 교차 참조

`iterate`(green 수렴 루프) · `tdd`/`test-gen`(경계 테스트) · `squad-plan`(분해·계획) · `squad-evaluator`(독립 평가) · `handoff`(세션 인계). 이 스킬은 그 위에서 절차를 지휘한다.

> 텔레메트리(opt-in): `source .claude/hooks/_metrics.sh 2>/dev/null && carve_metric workflow pass` (중단·실패로 끝나면 `fail`).
