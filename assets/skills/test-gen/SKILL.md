# test-gen — 테스트 생성

1. 대상의 공개 인터페이스와 UAT 수용 기준을 파악한다.
2. 정상·경계·실패 케이스를 각각 작성한다. 구현 결합 금지(동작을 검증).
3. 프로젝트 테스트 러너를 탐지해 사용한다. 생성 후 실행해 통과를 확인한다.

새 기능을 테스트 우선으로 만들 땐 `tdd` 스킬, 기존 코드 보강엔 이 스킬을 쓴다.
