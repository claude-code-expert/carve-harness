# iterate — 자율 수렴 루프

한 프롬프트로 "green까지 스스로 수렴"하는 primitive. 보고는 아래 규칙을 따른다.

## 루프

1. **목표 설정**: build → lint → test → typecheck를 돌려 green 기준을 잡는다(프로젝트에서 명령 탐지; 내장 `/verify`가 있으면 활용).
2. **실패 시 진단**: 에러 출력·스택트레이스를 읽고 근본 원인을 좁힌다(재현 → 격리 → 가설 → 증거 → 원인).
3. **최소 수정**: 원인에 대한 최소 변경만 적용한다. 투기적 변경 금지.
4. **재실행**: 1로 돌아가 다시 검증한다.

## 하드 스톱 (런어웨이 방지 — 필수)

- **최대 N회**(기본 5회) 또는 **무진전**(동일 실패 시그니처가 2회 연속) 시 즉시 중단한다.
- 중단 시 남은 실패와 막힌 지점(블로커)을 보고하고 사람의 판단을 요청한다. 무한 반복 금지.

## 보고

- 반복 과정을 내레이션하지 않는다. **최종 상태만**: 사용한 반복 횟수, 최종 pass/fail, 남은 실패(있으면).
- 루프 종료 시 텔레메트리 한 줄을 남긴다(opt-in일 때만 기록됨):
  `source .claude/hooks/_metrics.sh 2>/dev/null && carve_metric iterate pass`  (실패로 끝나면 `fail`).

## 격리 (선택)

- 위험하거나 큰 변경은 **git worktree**에서 루프를 돌려 진행 중 수정이 메인 트리를 흔들지 않게 한다(`parallel-agents` 참조).
  green이 된 뒤에만 병합한다.

> 범위: carve는 OS 샌드박스/컨테이너를 띄우지 않는다. 루프는 프로젝트 트리(선택적으로 worktree)에서 실행되며,
> 결정적 안전 게이트는 `pre-commit-lint`·`pre-push-test` 훅(exit 2)이 backstop으로 강제한다.
