# review — 코드 리뷰

1. `git diff`로 변경 범위를 본다. DIFF에 집중(전체 코드베이스 아님).
2. 심각도 순으로: Critical(보안·데이터손실) → High(로직·에러처리) → Medium(중복·네이밍) → Low(스타일).
3. 수정 코드 스니펫을 제시하되 직접 쓰지 않는다(읽기 전용). 깊은 리뷰는 `squad-review`에 위임.
