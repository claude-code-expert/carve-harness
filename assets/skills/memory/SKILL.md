# memory — 프로젝트 지속 메모리

코드/깃 히스토리로 알 수 없는 **비자명한 사실**만 저장한다.

- 한 파일 = 한 사실. 제목 + 왜 중요한지 + 어떻게 적용하는지.
- 코드 구조·과거 수정·CLAUDE.md가 이미 담은 것은 저장하지 않는다.
- 틀린 메모리는 지운다.

## 컨텍스트 다이어트 (세션 분리)
- 컨텍스트가 무거워지면(점유율 예산 초과 조짐) 현재 상태를 `handoff`로 스냅샷한 뒤,
  **컴포넌트·기능 단위로 새 세션 분리**를 제안한다(대화가 길어지면 앞 맥락 유지가 깨진다).
- 독립 작업은 git worktree로 분리할 수 있다(`parallel-agents` 참조).
