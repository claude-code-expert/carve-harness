# parallel-agents — 최소 병렬화로 최대 산출

> 임의의 "터미널 N개"를 좇지 않는다. 최소 병렬화로 최대 산출이 목표.

1. 독립적인 작업만 병렬화한다. 의존이 있으면 wave로 순차 진행한다.
2. 동시 **3~4개까지**. 각 에이전트는 단일 책임(Planner → Generator ⇄ Evaluator).
3. 코드 충돌 방지: 병렬 쓰기는 **git worktree**로 격리한다.
4. 1입력 → 1출력(파일). 에이전트 사이 컨텍스트는 분리하고, 결과는 오케스트레이터가 합친다.

주의: 병렬화에는 조율 비용이 있다. 독립성·격리가 없으면 순차가 낫다.
