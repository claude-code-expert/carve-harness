# tdd — Red-Green-Refactor

> mattpocock/skills(MIT)의 tdd 패턴에서 영감. 수직 슬라이스 방식.

1. **계획**: 인터페이스·우선순위 동작·테스트 범위를 먼저 합의한다(코드 전).
2. **트레이서 불릿**: end-to-end 테스트 1개로 접근을 증명하고 최소 구현으로 통과시킨다.
3. **증분 루프**: 테스트 하나 → 그 테스트만 통과할 최소 코드 → 다음. 투기적 기능 금지.
4. **리팩터는 GREEN 이후에만.** 테스트가 빨간 동안 리팩터하지 않는다.
5. 테스트는 공개 인터페이스로 **관측 가능한 동작**만 검증한다(구현 결합 금지). 명세처럼 읽혀야 한다.

금지: 모든 테스트를 먼저 쓰고 나서 모든 구현을 쓰는 horizontal 방식 — 상상한 동작을 검증하고 리팩터에 취약하다.
