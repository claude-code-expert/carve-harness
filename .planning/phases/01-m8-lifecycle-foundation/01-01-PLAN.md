<objective>
Establish the manifest schema v2 foundation: content-hash helper, the `ManifestFile` value object, a single backward-compat normalization point, and a disk migration. Every downstream lifecycle command (diff/update/migrate, installer, uninstall, doctor) consumes the normalized v2 shape — so this plan is the contract the rest of the phase builds on.

Purpose: Carry a per-file `{path, hash, assetVersion}` record so carve can later classify on-disk assets (unchanged / carve-updated / user-modified / new-recommended) and update them without clobbering user edits.
Output: `src/manifest.ts` schema v2 + `test/unit/manifest.test.ts`.
</objective>
