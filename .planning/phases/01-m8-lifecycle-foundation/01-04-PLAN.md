<objective>
Implement `carve update` and `carve migrate` — the lifecycle mutations. `update` consumes `classify` (01-03) to refresh only carve-updated assets in place, preserve user-modified assets (`.bak` once on a confirmed `--force` overwrite), and merely propose new-recommended assets. It audits the refreshed artifacts BEFORE writing and writes the manifest LAST (atomicity: a failed audit leaves the prior install + manifest untouched). `migrate` delegates to `migrateManifest` and prints the back-fill limitation.

Purpose: This is the user-facing payoff of M8 — updating a harness across carve versions without destroying user edits.
Output: `cmdUpdate`, `cmdMigrate` in `src/commands.ts`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| carve update → installed .claude/ tree | update mutates a partially user-owned tree; must not clobber user edits without consent and must be atomic on audit failure |
| generated update artifacts → disk | refreshed artifacts must pass the auditor before any write (no untrusted/over-permissioned asset lands) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | Tampering | update overwriting user-modified assets | mitigate | Skip user-modified by default; only --force + confirm overwrites; user content preserved as .bak once before overwrite (D6) |
| T-01-09 | Elevation of Privilege | refreshed update artifacts | mitigate | auditGate (secret/excess-perm/hook-injection/shell-syntax) runs BEFORE any write; blocked audit writes nothing (C5/D8) |
| T-01-10 | Tampering | partial update leaving inconsistent state | mitigate | Manifest written LAST via install(); .bak + file writes happen only after a passing audit so a failure leaves prior install + manifest unchanged (C7) |
| T-01-11 | Spoofing | update against un-migrated v1 manifest | mitigate | Empty-hash manifests are rejected with a `carve migrate` advisory before any classification-driven write (avoids mis-classifying every file as user-modified and force-clobbering) |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan. |
</threat_model>
