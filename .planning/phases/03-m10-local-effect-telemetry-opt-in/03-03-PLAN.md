    - Given a jsonl with several `{ts,hook,event}` lines, the per-hook output reflects correct total counts and the count of lines where `event === 'block'`.
    - Malformed lines (non-JSON, missing fields) are skipped defensively without crashing.
    - 0-fire detection: given a manifest whose files include an installed `carve-<id>.sh` hook that has zero metric lines, that hook id is listed as 0-fire; with no manifest the 0-fire section is skipped.
  </behavior>
  <action>
Add `export function cmdReport(root: string, io: IO): number`:
- `const metricsPath = join(root, '.claude/.carve-metrics.jsonl');` If `!existsSync(metricsPath)` → log the Korean "기록 없음" message above and `return 0`.
- Read the file, split on `/\r?\n/`, filter blank lines, and for each line `try { JSON.parse(line) } catch { continue }` (skip malformed). Validate the parsed object has string `hook` and `event`; skip otherwise.
- Aggregate into a `Map<string, { total: number; blocks: number }>` keyed by `hook`; increment `total` always and `blocks` when `event === 'block'`.
- 0-fire: `const m = readManifest(root);` if `m`, collect installed carve hook ids from `m.files` where `path` matches `^\.claude/hooks/carve-(.+)\.sh$` (capture the id; strip the `carve-` prefix and `.sh` suffix). Any captured id with no entry in the aggregation Map → 0-fire (noise candidate). If `m` is null, skip the 0-fire section.
- Output a Korean summary via `io.log`: a header line, one line per hook with fire/block counts (e.g. `  · <hook>: fire <total> · block <blocks>`), a `0회 발화(노이즈 후보): ...` line (or "없음"), and a total line. `return 0`.
- Keep it simple — no sorting requirement beyond deterministic iteration; aggregation Map insertion order is fine. Use only `io.log`/`io.error`. Reuse already-imported fs helpers and `readManifest` (already imported in this file).
  </action>
